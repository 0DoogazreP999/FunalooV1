# Funaloo NEXUS-116 Engine: A Meta-Learning, Self-Evolving System

The Funaloo NEXUS-116 engine has been upgraded into a true meta-learning system. This document details the dual-loop feedback mechanism that allows the engine to not only learn from its generations but also learn how to learn better, creating a continuous cycle of improvement.

## Key Features

### 1. Enriched, Agent-Specific Knowledge Capture

The AI agents now enrich their knowledge capture by tagging insights with their specific roles. The `INSIGHT_BLOCK` in each generated file now includes the `agent`'s name, allowing for granular tracking of where each piece of knowledge originates.

### 2. Structured, Agent-Centric Knowledge Base

The `KnowledgeExtractionPlugin` has been enhanced to create a more structured `knowledge.json` file. Instead of a flat list, the knowledge is now organized by the agent that generated it. This creates a powerful, queryable knowledge base that maps insights to their domain of origin.

### 3. Contextual Knowledge Injection

The generation pipeline has been upgraded to perform **Contextual Knowledge Injection**. The system now intelligently injects tailored information:

*   **Architectural Context:** All agents receive the high-level architectural knowledge generated by the `architect` agent.
*   **Specialized Learning:** Each agent receives all the specific knowledge from its *own* previous generations.

### 4. Meta-Learning with the `Evolver` Agent

A new, meta-level agent, the **`Evolver`**, has been introduced. The `Evolver` acts as a "prompt engineer," analyzing the collected knowledge in `knowledge.json` and proposing improvements to the system prompts of the other agents. This creates a second-order feedback loop where the engine actively refines its own internal learning processes.

## The Dual Self-Evolution Loop

This creates a powerful, dual-loop self-evolution mechanism:

1.  **Generate & Tag:** Agents generate code and tag their insights with their agent name.
2.  **Extract & Structure:** The `KnowledgeExtractionPlugin` extracts these insights and organizes them by agent in `knowledge.json`.
3.  **Evolve Prompts (Meta-Learn):** The `Evolver` agent analyzes `knowledge.json` and generates `prompt-evolution.json`, a file containing suggested improvements to the other agents' system prompts.
4.  **Apply Evolutions:** At the start of the next generation, the engine reads `prompt-evolution.json` and dynamically applies the new, evolved prompts to the agents.
5.  **Learn Contextually & Improve:** The agents, now with evolved prompts, receive their contextual knowledge and generate the next iteration of code, which is more informed, more specialized, and of higher quality.

This dual-loop feedback mechanism transforms the Funaloo NEXUS-116 engine into a true meta-learning system. It doesn't just learn; it learns *how to learn better*, leading to an exponential increase in capability and a truly endless potential for game generation.
